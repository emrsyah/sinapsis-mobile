# Sinapsis Mobile

Academic note-taking mobile app with AI-powered study tools.

Built with Flutter 3.38+ / Dart 3.7+, using Riverpod for state management, GoRouter for navigation, and Freezed for immutable models. Backend is a Laravel API.

---

## Prerequisites

| Tool | Version |
|------|---------|
| Flutter SDK | >= 3.38.0 |
| Dart SDK | >= 3.7.0 |
| Android Studio / VS Code | Latest |
| Firebase CLI (for push notifications) | `dart pub global activate flutterfire_cli` |

## Quick Start

```bash
# 1. Clone and install dependencies
git clone <repo-url>
cd sinapsis_mobile
flutter pub get

# 2. Run code generation (required for Riverpod + Freezed models)
dart run build_runner build --delete-conflicting-outputs

# 3. Run on connected device / emulator
flutter run
```

Available devices can be listed with `flutter devices`. To run on a specific target:

```bash
flutter run -d chrome     # web
flutter run -d windows    # desktop
flutter run -d <device>   # android / ios emulator
```

### Firebase Setup (for push notifications)

Firebase is optional for UI development but required for push notifications.

```bash
# Install Firebase CLI
dart pub global activate flutterfire_cli
flutterfire login

# Configure Firebase (generates firebase_options.dart automatically)
flutterfire configure
```

This will:
- Create a Firebase project (or link to existing)
- Place `google-services.json` in `android/app/`
- Place `GoogleService-Info.plist` in `ios/Runner/`
- Generate `lib/firebase_options.dart` with your project config

Without this, the app will throw on startup at `Firebase.initializeApp()`. To skip Firebase temporarily, comment out the Firebase + notification init lines in `main.dart`.

### Environment Variables

API URLs and keys are configured via `--dart-define` at build time (see `lib/core/constant.dart`):

```bash
flutter run \
  --dart-define=API_BASE_URL=https://api.sinapsis.app \
  --dart-define=REVERB_WS_URL=wss://api.sinapsis.app \
  --dart-define=REVERB_KEY=your_key \
  --dart-define=UPLOADTHING_API_KEY=your_key
```

Defaults are in `lib/core/constant.dart` — works without any defines for local dev if the backend is running at `api.sinapsis.app`.

## Architecture Overview

```
Screen (ConsumerWidget)
  → watches Riverpod provider (AsyncValue state)
  → calls provider method on user interaction
    → provider calls repository
      → repository calls ApiClient (Dio)
        → response deserialized to Freezed model
      → provider updates AsyncValue
  → screen rebuilds via ref.watch()
```

### Key rules
- **Screens** never call `ApiClient` directly — always through a repository
- **Features** never import from other features' internal files — only from `models/`, `core/`, `shared/`
- **Navigation** is handled entirely by GoRouter — never use `Navigator.push` directly
- **Models** must use Freezed — no plain Dart classes for API data
- **Re-fetch after mutations** via `ref.invalidate()` — never manage lists manually

## Code Generation

This project uses `build_runner` for code generation. You need to run it after:
- Adding/modifying `@riverpod` annotations
- Changing `@freezed` model fields
- Any `.g.dart` or `.freezed.dart` file is stale

```bash
# One-time build
dart run build_runner build --delete-conflicting-outputs

# Watch mode (auto-rebuilds on file changes)
dart run build_runner watch --delete-conflicting-outputs
```

### Provider naming convention

With `riverpod_annotation` v4, the generated provider name is derived from the class:

| Class name | Generated provider |
|---|---|
| `AuthNotifier` | `authProvider` |
| `NoteListNotifier` | `noteListProvider` |
| `NoteDetailNotifier` | `noteDetailProvider` |
| Function `noteDetail()` | `noteDetailProvider` |

The pattern is: strip the `Notifier`/`Provider` suffix (if present), then append `Provider`.
